CHESS

Game Object
  - creates a board object to hold game state
  - handles player input
  - handles "graphics"; interprets the Board array

Board Object
  - represented by a multi-d array
  - you can query the board to find out what's at a certain position
  - validates moves (e.g. if a move is outside or a piece is already there)

Piece Object
  - superclass of all pieces
  - holds the piece's position

Knight Object, etc.
  - holds the array of possible moves (x+2,y+1, etc)
  - holds any other piece logic (e.g. pawns can sometimes move diagonally)
